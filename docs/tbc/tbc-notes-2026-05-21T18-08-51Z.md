## Gotchas not yet in CLAUDE.md or memory

- **Re-inlining `plan.css` / `plan.js` is a multi-file operation.** If we change either canonical file, we must re-inline into `plan-template.html` and all three `EXAMPLE-*.html` files. Use the Python regex approach from this session (find `<style id="plan-css-inline">...</style>` and `<script id="plan-js-inline">...</script>`, replace contents in one shot). A one-shot Edit on each file will not scale.
- **Copy diff paste-backs may contain em-dashes** because Scott's browser tab can be running an older cached version of plan.js. When applying his edits, silently convert any em-dashes in his new text to colon, period, or parens per the standing rule. Do not preserve em-dashes just because they came back in the diff.
- **Bulk em-dash to colon substitution can create "bracket-colon" issues** where colons end up doing parenthetical work that was originally done by em-dashes around a list (e.g., `Python: LangChain, X, Y: share one property`). Fix by using parentheses around the bracketed list instead of colons. Already cleaned in the portfolio plan; watch for this when authoring any new doc.
- **`<a class="xref">` does not include the `§` in the link text.** The `.xref` CSS adds the prefix. If you literally type `<a class="xref">§Foo</a>` you will get `§§Foo` rendered. The skill's own DESIGN.md is clear on this; I introduced the bug in the first plan draft and Scott's edits surfaced it. Same applies to `<a class="xref doc">` and `⧉`.
- **`dist/atelier-plans/` is the install source; `.claude/skills/atelier-plans/` is canonical.** They are intentionally out of sync after the em-dash scrub (Scott chose "leave dist alone"). If we ever want to publish or reinstall the skill, sync dist forward; otherwise ignore it.
- **The Bluesky pivot may force a re-think of L1's primary analog.** `kytmanov/obsidian-llm-wiki-local` extracts from markdown. Bluesky posts come from an API, not a vault. The extraction pattern still applies, but the input adapter changes. Worth a short re-analog hunt focused on "Bluesky data ingestion in Python" once the pivot lands.
