## Gotchas not yet in CLAUDE.md or memory

- **Plan anchor IDs are deliberately decoupled from H3 text.** The Levels section renames each `<h3>` to "Branch 1 · `feat/transcripts`" etc. but keeps the original anchor IDs (`#l1`–`#l10`). This preserves all the xrefs from invariants, ADRs, and patterns that reference `#l1`–`#l10`. Do not rename the anchor IDs even though the prose now says "branch 1."
- **`<a class="xref">` CSS auto-prefixes with §.** If you write `<a class="xref" href="#l3">branch 3</a>` it renders as "§branch 3" which reads as "section branch 3" — awkward. Prefer "Branch <a class=\"xref\" href=\"#l3\">3</a>" (renders "Branch §3") or just plain `<a href="#l3">branch 3</a>` (no §). The eval-strategy ordered list uses the former pattern; the cross-cutting patterns section uses the bare `L3`-as-link-text pattern. Both work; do not "fix" by changing the label to "branch 3" inside an existing xref-classed anchor.
- **Pydantic-ai's `direct` module IS the "Responses helper" Scott was thinking of.** Functions are `pydantic_ai.direct.model_request` (async) and `model_request_sync`. Same model-string format. Supports tool calls via `ModelRequestParameters.function_tools` if you want. No agent loop. The library docs explicitly position it as "a simpler interface compared to the Agent API when full functionality is not required."
