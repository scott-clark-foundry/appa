## Gotchas not yet in CLAUDE.md or memory

- **Background-shell PATH is stripped.** The harness's background Bash ships a minimal PATH and `export PATH=…` does not carry across statements; subprocesses cannot resolve `python3`, `claude`, or even `date` as bare words. The fix is to pass env directly to the subprocess: `/usr/bin/env PATH=… /opt/homebrew/bin/python3 …`. Cost three retries this session. If a future background command exits 127 with a `command not found`, this is the first place to look.
- **skill-creator's description optimizer hits a ~50% recall floor on imperative trigger queries.** All five descriptions it tested scored perfect precision but 4/8 test recall, and no rewrite across 15 candidates broke the ceiling. The skill-creator README names this behavior: Claude often won't consult a skill for short imperative requests it believes it can handle directly. Not a wording defect; don't chase it.
- **Project-local and plugin-installed skills with the same name coexist.** The seven superpowers skills we copied live alongside the plugin install of superpowers. Per harness behavior project-local usually wins, but if the wrong version triggers, this is the place to look. Scott has not chosen between uninstalling the plugin or renaming the local copies; he was OK leaving it for now.
- **MIT attribution risk in the public repo.** The four moved skills under `assistant/.claude/skills/` came from MIT-licensed superpowers but did not bring a LICENSE notice with them. When `assistant/` is first pushed public, including MIT attribution for the copied skills (either a top-level NOTICES file or inline frontmatter) avoids a quiet license-compliance gap. The Visual Companion's `frame-template.html` does still link to `obra/superpowers` as attribution; the others do not.
